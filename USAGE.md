## 友情連結功能

友情連結功能基於 WordPress 內置的連結管理器實現，可以幫助你管理和展示網站的友情連結。

### 啟用連結管理器

WordPress 在較新版本中預設隱藏了連結管理器功能。如果你在 WordPress 後臺左側功能表中沒有看到「連結」選項，需要先啟用它：

**方法1：使用外掛程式（推薦）**
1. 在 WordPress 後臺進入 `外掛程式` → `安裝外掛程式`
2. 搜尋 `Link Manager` 或 `Links Manager`
3. 安裝並啟用外掛程式
4. 重新整理頁面後，左側功能表會出現 `連結` 選項

**方法2：新增代碼**
在主題的 `functions.php` 檔或使用代碼片段外掛程式新增以下代碼：

```php
add_filter('pre_option_link_manager_enabled', '__return_true');
```

### 新增友情連結

1. 在 WordPress 後臺左側功能表中點選 `連結` → `新增`
2. 填寫連結資訊：
   - **名稱**：友情連結網站的名稱
   - **網址**：友情連結網站的 URL（必填）
   - **描述**：網站的簡介描述（可選）
   - **類別**：為連結選擇或建立分類（可以建立多個分類來組織連結）
3. 進階選項：
   - **連結圖像**：可以設定網站的 Logo 或圖示 URL
   - **連結目標**：通常選擇 `_blank`（在新視窗打開）
   - **連結關係 (rel)**：可以設定為 `nofollow` 或 `friend`
4. 點選 `新增連結` 按鈕儲存

**管理連結分類：**
- 點選 `連結` → `連結分類` 可以建立和管理連結分類
- 建議建立如 "推薦連結"、"合作夥伴"、"個人博客" 等分類來組織你的友情連結

### 建立友情連結頁面

1. 在 WordPress 後臺進入 `頁面` → `新建頁面`
2. 輸入頁面標題，如 "友情連結"
3. 在右側的 `頁面屬性` → `範本` 下拉式功能表中選擇 **`友情連結`** 範本
4. 在頁面編輯器中可以輸入友情連結頁面的說明文字（可選）
5. 在頁面下方的 `自訂欄位` 或 `頁面設定` 區域（根據主題版本可能位置不同）：
   - **page_links_id**：選擇要顯示的連結分類 ID（可多選）
   - **use_theme_link_forward**：是否使用主題的跳轉功能（用於統計和保護）
6. 點選 `發佈` 按鈕

**取得連結分類 ID 的方法：**
1. 進入 `連結` → `連結分類`
2. 滑鼠懸停在分類名稱上
3. 在瀏覽器底部狀態列或檢視連結位址，可以看到類似 `tag_ID=123` 的 ID 數位
4. 記錄下這個 ID 數位，在頁面設定中使用

### 首頁顯示友情連結

如果想在首頁或其他位置顯示友情連結模組：

1. 進入 Puock 主題設定後臺
2. 找到 `首頁設定` 或 `全域設定`
3. 查找 `友情連結` 相關設定：
   - **index_link_id**：選擇要在首頁顯示的連結分類 ID
   - **index_link_order**：排序方式（ASC 昇冪 / DESC 降冪）
   - **index_link_order_by**：排序依據（link_id / link_name / link_rating 等）
   - **link_page**：指定友情連結完整頁面的 ID（顯示"更多連結"按鈕）
4. 儲存設定

## Puock提示框功能

Puock 主題提供了多種樣式的提示框短代碼，可以在文章或頁面中使用。

### 基礎用法

在文章編輯器中使用以下格式：

```
[t-類型]這裡是提示框的內容[/t-類型]
```

**示例：**

```
[t-primary]這是一個主要提示框[/t-primary]
```

效果：顯示一個藍色背景的提示框

### 可用類型

Puock 主題支援以下 6 種提示框類型：

| 短代碼 | 類型 | 顏色 | 適用場景 |
|--------|------|------|----------|
| `[t-primary]` | 主要 | 藍色 | 一般資訊提示 |
| `[t-success]` | 成功 | 綠色 | 成功消息、正面資訊 |
| `[t-info]` | 資訊 | 淺藍色 | 補充資訊、提示說明 |
| `[t-warning]` | 警告 | 黃色 | 警告資訊、注意事項 |
| `[t-danger]` | 危險 | 紅色 | 危險警告、錯誤資訊 |
| `[t-dark]` | 深色 | 深灰色 | 次要資訊、引用內容 |

**使用示例：**

```
[t-success]操作成功！您的設定已儲存。[/t-success]

[t-warning]注意：此操作不可撤銷，請謹慎操作。[/t-warning]

[t-danger]錯誤：檔上傳失敗，請檢查檔大小和格式。[/t-danger]

[t-info]提示：您可以在設定中修改預設選項。[/t-info]
```

### 新增圖示

可以在提示框中新增 Font Awesome 圖示：

```
[t-primary icon="fa fa-info-circle"]帶圖示的提示框內容[/t-primary]
```

**常用圖示示例：**

```
[t-success icon="fa fa-check-circle"]成功完成任務[/t-success]

[t-warning icon="fa fa-exclamation-triangle"]請注意操作風險[/t-warning]

[t-danger icon="fa fa-times-circle"]發生嚴重錯誤[/t-danger]

[t-info icon="fa fa-lightbulb"]這是一個小技巧[/t-info]
```

更多圖示可以在 [Font Awesome 官網](https://fontawesome.com/icons) 查找。

### 輪廓樣式

使用 `outline="true"` 參數可以啟用輪廓樣式（邊框樣式，無背景填充）：

```
[t-primary outline="true"]這是輪廓樣式的提示框[/t-primary]
```

**完整示例（帶圖示和輪廓）：**

```
[t-info outline="true" icon="fa fa-star"]這是一個帶星標圖示的輪廓提示框[/t-info]
```

## 其他短代碼功能

Puock 主題還提供了豐富的其他短代碼功能：

### 回覆可見

在文章中隱藏部分內容，需要讀者評論後才能檢視：

```
[reply]
這裡的內容需要評論後才能檢視
支援多行內容
[/reply]
```

### 登入可見

內容僅登入使用者可見：

```
[login]
這裡的內容僅登入使用者可見
[/login]
```

### 登入並驗證信箱可見

內容需要登入且驗證過信箱的用戶才能檢視：

```
[login_email]
這裡的內容需要登入並驗證信箱後才能檢視
[/login_email]
```

### 密碼可見

使用密碼保護內容：

```
[password pass="123456" desc="請輸入密碼檢視隱藏內容"]
這裡是需要密碼才能檢視的內容
[/password]
```

參數說明：
- `pass`：設定密碼（必填）
- `desc`：提示文字（可選，預設："此處含有隱藏內容，需要正確輸入密碼後可見！"）

### 折疊面板

建立可折疊展開的內容：

```
[collapse title="點選展開檢視詳細內容"]
這裡是折疊的內容
可以是很長的文本
[/collapse]
```

### 文件下載

顯示檔下載資訊框：

```
[download file="軟體名稱-v1.0.zip" size="25.6MB"]
https://example.com/download/file.zip
[/download]
```

參數說明：
- `file`：檔案名
- `size`：文件大小

### 視訊播放

嵌入視訊播放機：

```
[video url="https://example.com/video.mp4" autoplay="false" pic="https://example.com/poster.jpg"]
```

參數說明：
- `url`：視訊位址（必填）
- `autoplay`：是否自動播放（true/false，預設 false）
- `type`：視訊類型（auto/hls/flv 等）
- `pic`：視訊封面圖片
- `ssl`：是否強制使用 HTTPS（true/false）

### 音訊播放

嵌入音訊播放機：

```
[music]https://example.com/audio.mp3[/music]
```

### GitHub 卡片

顯示 GitHub 倉庫資訊卡片：

```
[github]用戶名/倉庫名[/github]
```

**示例：**

```
[github]Licoy/wordpress-theme-puock[/github]
```

## 使用技巧

### 1. 古騰堡編輯器中使用短代碼

- 新增一個 `短代碼` 區塊
- 在區塊中輸入短代碼
- 預覽或發佈後即可看到效果

### 2. 經典編輯器中使用短代碼

- 切換到 `文本` 模式（而非 `視覺化` 模式）
- 直接輸入短代碼
- 切換回 `視覺化` 模式預覽效果

### 3. 使用 Puock 提示框區塊（古騰堡）

如果使用古騰堡編輯器，可以直接搜尋並新增 `Puock提示框` 區塊：
1. 點選 `+` 新增區塊
2. 搜尋 `Puock` 或 `提示框`
3. 選擇 `Puock 提示框` 區塊
4. 在區塊設定中配置顏色、文本、圖示等

### 4. 短代碼嵌套

短代碼支援嵌套使用：

```
[collapse title="檢視更多資訊"]
[t-info]這是折疊面板中的提示框[/t-info]
這是一些文字內容
[t-warning]這是警告提示[/t-warning]
[/collapse]
```

### 5. 自訂樣式

可以為提示框新增自訂 CSS 類：

```
[t-primary class="my-custom-class"]帶自訂樣式的提示框[/t-primary]
```

然後在主題的自訂 CSS 中新增樣式：

```css
.my-custom-class {
    font-size: 18px;
    border-radius: 10px;
}
```

## 常見問題

### Q1: 為什麼看不到「連結」功能表？

A: WordPress 在新版本中預設隱藏了連結管理器，需要安裝 Link Manager 外掛程式或新增代碼啟用。詳見 [啟用連結管理器](#啟用連結管理器)。

### Q2: 友情連結頁面顯示為空白？

A: 檢查以下幾點：
1. 是否已新增友情連結並設定為可見
2. 頁面範本是否選擇了「友情連結」
3. 頁面的 `page_links_id` 欄位是否正確設定了分類 ID
4. 連結分類 ID 是否正確

### Q3: 短代碼不起作用，直接顯示代碼文本？

A: 可能原因：
1. 短代碼格式錯誤（檢查方括號和標籤名稱）
2. 在代碼塊中使用了短代碼（短代碼在代碼塊中不會被解析）
3. 主題或外掛程式衝突（嘗試切換到預設主題測試）

### Q4: 提示框樣式不對或不顯示？

A: 檢查：
1. 主題版本是否是最新版
2. 瀏覽器快取是否清除
3. 短代碼類型名稱是否正確（如 `t-primary` 而不是 `t-blue`）

### Q5: 如何取得連結分類的 ID？

A: 進入 `連結` → `連結分類`，滑鼠懸停在分類名稱上，檢視瀏覽器底部或位址欄中的 `tag_ID=數位`，這個數位就是分類 ID。
